tThresholdViolationAlert

Warning

This component will be available in the Palette of the studio on the condition that you have subscribed to any Talend Platform product.

Warning

This component is only usable with the report database (MySQL) that stores the results of the analyses executed in the studio. For information on the report database, see the report chapter in Talend Data Quality Studio User Guide.

tThresholdViolationAlert properties

Component family

Data Quality

 

Function

tThresholdViolationAlert can alert to any threshold violations regarding the thresholds set on indicators in different analyses created in the Profiling perspective of the studio.

Purpose

tThresholdViolationAlert communicates with the report database that stores the analysis results. It scans the report database looking for the thresholds that are violated in the column analyses in the selected report. You can then use it with other components in the job, for example tSendMail, to alert to any threshold violations regarding the thresholds set on indicators used in the column analyses.

In order not to create such a job manually from scratch, you can generate a ready-to-use job that uses tThresholdViolationAlert from the Profiling perspective of the studio. For further information, see the report chapter in Talend Data Quality Studio User Guide.

Basic settings

Schema and Edit schema

A schema is a row description, i.e., it defines the number of fields to be processed and passed on to the next component. The schema is either built-in or remotely stored in the Repository.

 

 

Built-in: You create and store the schema locally for this component only. Related topic: see Talend Data Integration Studio User Guide.

 

 

Repository: You have already created the schema and stored it in the Repository and thus reuse it when needed. Related topic: see Talend Data Integration Studio User Guide.

 

Output field

Select in the list the output field that will hold the input data.

 

Additional Information

When the job is generated, this area is prefilled with detail information about the threshold violations in the selected report. It lists the values of the indicators which do not respect the thresholds. It also lists the report name, the analysis name and the analyzed element description (column name, table name and schema name).

This same information will be sent in the email message.

Usage

This component is used as an intermediate component in a Job design. It needs an input and output components. tThresholdViolationAlert only works with a MySQL database.
