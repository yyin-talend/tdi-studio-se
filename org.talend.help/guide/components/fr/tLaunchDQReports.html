tLaunchDQReports

Avertissement

Ce composant est disponible dans la Palette du studio si vous avez souscrit à l'un des produits Talend Platform.

Propriétés du tLaunchDQReports

Famille de composant

Data Quality

 

Fonction

Le tLaunchDQReports lance des rapports existants dans le studio de Talend Data Quality sans avoir besoin d’ouvrir la plateforme . Il fait la même action que l’objet Generate report file du menu de la plateforme Talend Data Quality.

Objectif

Le tLaunchDQReports peut communiquer avec Talend Data Quality pour lancer une analyse et sauvegarder les résultats de l’analyse dans une base de données. En utilisant ce composant, vous pouvez lancer la génération de tous types de rapports : PDF, XML, ou HTML.

Pour plus d’informations sur la génération d’un fichier de rapport, consultez le Guide utilisateur de Talend Data Quality.

Basic settings

Talend Data Quality executable file

Chemin d’accès au fichier exécutable du studio.

 

Project Directory

Saisissez ou copiez-collez le chemin d’accès au projet auquel appartient le rapport qui vous intéresse.

 

Report filenames

Copiez le chemin d’accès technique du rapport à générer depuis le studio de Talend Data Quality et copiez-le dans ce champ. Vous pouvez paramétrer différents chemins d’accès en les séparant par une virgule.

Note

Vous pouvez afficher le chemin d’accès technique du rapport que vous souhaitez générer, dans la zone Detail view sous l’arborescence DQ Repository dans le studio Talend Data Quality, quand vous sélectionnez le rapport correspondant dans l’arborescence.

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les métadonnées de process du Job, aussi bien au niveau du Job qu’au niveau de chaque composant.

Utilisation

Ce composant est utilisé en standalone. Il peut lancer des rapports depuis le studio de Talend Data Quality.

Note

Il est recommandé de générer automatiquement un Job de lancement de rapport utilisant ce composant à partir de la vue DQ Repository dans la perspective Integration de votre Studio. Pour plus d’informations concernant la génération automatique d’un Job, consultez le Guide utilisateur de Talend Data Quality.

Avertissement

Pour qu’il soit exécutable, un Job contenant ce composant doit avoir un accès au fichier exécutable du studio Talend. Si le Job est utilisé en Jobserver, il nécessitera un accès au fichier exécutable du studio Talend.

Scénario : Lancer à distance un rapport d’une application Web Talend Administration Center donnée

Ce scénario illustre les opérations collaboratives effectuées pour exécuter à distance un Job de génération de rapport depuis une application Web Talend Administration Center. Pour ce faire, vous devez commiter toutes les métadonnées requises dans le référentiel SVN utilisé par l’application Web Talend Administration Center et installer un serveur d’exécution dans votre système.

Prérequis

Avant de commencer ce scénario, procédez comme suit afin de définir les prérequis :

  1. Créez un projet dans votre application Web Talend Administration Center. Les opérations collaboratives pour lancer un rapport seront effectuées dans ce projet. Dans ce scénario, le projet est appelé test_ychen.

    Pour plus d’informations concernant la création d’un projet dans l’application Web Talend Administration Center, consultez le Guide utilisateur de Talend Administration Center.

    Note

    Si vous n’avez pas les droits requis pour créer ou gérer un projet, contactez l’administrateur de votre application Web Talend Administration Center.

  2. Etablissez une connexion à partir de votre Studio au référentiel distant stockant le projet susmentionné.

    Pour plus d’informations concernant la création d’une connexion de ce type, consultez le Guide utilisateur de Talend Data Integration Studio.

  3. Lancez votre Studio avec cette nouvelle connexion et le nouveau projet créé. pour plus d’informations concernant un tel lancement du Studio, consultez le Guide utilisateur de Talend Data Integration Studio.

  4. Dans la perspective Profiling de votre Studio, créez un rapport qui sera lancé et généré à distance dans ce scénario. Dans ce scénario, ce rapport est US_State.

    Pour plus d’informations concernant la création d’un rapport dans la perspective Profiling et la configuration des paramètres relatifs à la génération, consultez le Guide utilisateur de de Talend Data Quality.

  5. Dans l’éditeur de rapport de la perspective Profiling, ajoutez l’analyse, US_State dans ce scénario, sur laquelle vous souhaitez baser votre rapport et saisissez les paramètres afin de générer ce rapport au format PDF et de le stocker dans le répertoire suivant :

    C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center.

    Pour plus d’informations concernant la configuration des paramètres relatifs à la génération de rapports, consultez le Guide utilisateur de Talend Data Quality.

    Note

    Si vous n’avez pas créé de base de données de rapports pour garder l’historique des rapports, votre Studio vous demande d’en créer une. Pour plus d’informations, consultez le Guide utilisateur de Talend Data Quality.

    Ainsi, les métadonnées requises, comme les analyses, les rapports et les connexions aux bases de données ont été commitées dans le référentiel SVN utilisé par votre application Web Talend Administration Center.

  6. Dans le système où est votre Studio, installez et lancez un JobServer. Pour plus d’informations, consultez le Guide d’installation et de migration Talend.

Lancer un rapport à distance

Après avoir terminé les prérequis, procédez comme suit pour reproduire ce scénario :

  1. Dans la vue DQ Repository de la perspective Profiling, cliquez-droit sur le rapport US_State sous le nœud Report et sélectionnez Generate Jobs > Launch a report dans le menu contextuel. Cela crée un Job de génération de rapport.

    Note

    Il est recommandé de vérifier les propriétés (Basic settings) du composant tLaunchDQReports utilisé dans ce Job afin de s’assurer que les chemins d’accès générés pointent bien vers le fichier exécutable de votre Talend Data Quality, vers le rapport qui vous intéresse et vers le projet auquel appartient ce rapport.

  2. Dans l’arborescence de votre Application Web Talend Administration Center, développez Job Conductor et cliquez sur Servers pour afficher la liste des serveurs distants d’exécution disponibles pour déployer et exécuter les Jobs.

  3. Dans la barre d’outils de la page Servers, cliquez sur Add pour vider les champs du panneau de configuration Execution server.

    Dans ce panneau de configuration, saisissez et sauvegardez les informations de configuration du JobServer que vous avez installé et lancé dans votre système.

    Pour plus d’informations concernant la configuration requise pour vous connecter à un JobServer, consultez le Guide utilisateur de Talend Administration Center.

    Dans ce scénario, le libellé (Label) du JobServer est Report_Server, son hôte (Host) est 10.42.20.246 et les ports relatifs sont 8000, 8001, et 8888.

    Cliquez sur Save pour valider cette configuration. Le nouveau serveur s’affiche dans la liste des serveurs dans la page Servers et la coche verte indique que la connexion au serveur est établie.

  4. Dans la vue Menu de votre application Web Talend Administration Center, développez Job Conductor et cliquez sur Execution task pour afficher la liste des tâches dans la page Job Conductor.

  5. Dans la barre d’outils de la page Job Conductor, cliquez sur Add pour vider les champs du panneau de configuration Execution task.

    Dans ce panneau de configuration, saisissez les paramètres requis pour l’exécution du Job de génération de rapport que vous avez créé.

    • Dans le champ Label du panneau Execution task, saisissez le nom de la tâche qui sera affichée dans la liste des tâches d’exécution.

    • Dans le champ Project, sélectionnez le projet dans lequel a été créé le Job de génération de rapport. Dans ce scénario, le projet est test_ychen.

    • Dans le champ Branch, sélectionnez la branche du projet, trunk dans ce scénario.

      Dans le champ Job, sélectionnez le Job de génération de rapport à utiliser.

    • Dans les champs Version et Context, sélectionnez les informations correspondant au Job. Dans ce scénario, les valeurs 0.1 et Default sont utilisées.

    • Dans le champ Execution server, sélectionnez le JobServer qui exécutera la tâche. Ce serveur doit être installé dans votre système. Dans ce scénario, le serveur est Report_Server, que vous avez ajouté à la liste des serveurs.

  6. Cliquez sur Save pour valider cette configuration. La nouvelle tâche apparaît dans la liste des tâches dans la page Job Conductor.

  7. Dans la liste des tâches, sélectionnez la nouvelle tâche.

  8. Dans la barre d’outils du Job Conductor, cliquez sur Generate.

  9. Une fois le statut de la tâche sur Ready to send dans la liste des tâches, sélectionnez à nouveau cette tâche et cliquez sur Deploy dans la barre d’outils de la page Job Conductor.

  10. Une fois le statut de la tâche sur Ready to run dans la liste des tâches, sélectionnez à nouveau cette tâche et cliquez sur Run dans la barre d’outils de la page Job Conductor.

Une fois l’exécution du Job effectuée, vérifiez dans le dossier de sortie défini dans la vue Generated Report Settings de l’éditeur de rapport dans la perspective Profiling de votre Studio.

Dans ce scénario, le chemin d’accès au dossier de sortie est :

C:\Documents and Settings\ychen\Bureau\TEST_YCHEN\Report_Center.
