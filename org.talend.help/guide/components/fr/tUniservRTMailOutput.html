Propriétés du tUniservRTMailOutput

Famille de composant

Data Quality

 

Fonction

Le tUniservRTMailOutput met à jour le pool d'index utilisé pour les recherches de doublons.

Objectif

La tUniservRTMailOutput garde le pool d'index synchronisé.

Basic settings

Schema et Edit schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (Built-in) soit distant dans le Repository.

 

Host name

Nom de l'hôte du serveur, entre guillemets doubles.

 

Port

Numéro du port d'écoute du serveur, entre guillemets doubles.

 

Service

Le nom du service est "mail" par défaut. Saisissez une nouveau nom si nécessaire (si le suffixe du service change, par exemple), entre guillemets doubles.

 

Action on data

Opérations réalisables dans le pool d'index :

Insert : insère un nouvel enregistrement dans le pool d'index. Cette requête échoue si l'enregistrement possédant la même référence existe déjà dans le pool.

Update : met à jour un enregistrement existant dans le pool. Cette requête échoue si aucun enregistrement ne possède le nom de la référence donnée.

Insert or update : insère un nouvel enregistrement dans le pool d'index. Si l'enregistrement avec la référence donnée existe déjà, une mise à jour est effectuée.

Update or insert : met à jour l'enregistrement avec la référence donnée. Si l'enregistrement n'existe pas dans le pool d'index, un nouvel enregistrement est inséré..

Delete : supprime l'enregistrement correspondant à la référence donnée dans le pool d'index.

Advanced settings

Uniserv Parameters

Pour plus d'informations, consultez le manuel utilisateur d'Uniserv mailRetrieval.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données d'identification au niveau du Job et du composant.

Utilisation

Le tUniservRTMailOutput met à jour les pool d'index et passe l'ensemble d'entrée. La sortie contient le statut de l'opération. Si l'opération échoue, un message d'erreur message s'affiche.

Limitation

Pour utiliser le tUniservRTMailOutput, le logiciel Uniserv mailRetrieval doit être installé.

Note

Avant d'utiliser le tUniservRTMailOutput pour la première fois, un pool d'index doit être créé. Créez-en un à l'aide du composant tUniservRTMailBulk.

Scénario associé

Pour un scénario associé, consultez la section intitulée « Scénario : Ajouter des contacts au pool d'index de mailRetrieval ».
