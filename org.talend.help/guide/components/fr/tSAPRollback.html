tSAPRollback

Propriétés du tSAPRollback

Ce composant est étroitement lié aux composants tSAPCommit et tSAPConnection. Ils sont généralement utilisés ensemble lors de transactions.

Famille de composant

Business/SAP

 

Fonction

Le composant tSAPRollback annule la transaction dans le SAP connecté.

Objectif

Le tSAPRollback évite le commit de transaction involontaire.

Basic settings

SAPConnection Component list

Sélectionnez le composant de connexion tSAPConnection dans la liste si vous prévoyez d’ajouter plus d’une connexion à votre Job en cours.

 

Release Connection

Décochez cette case afin de continuer à utiliser la connexion sélectionnée, une fois que le composant a exécuté sa tâche.

Advanced settings

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau du composant.

Dynamic settings

Cliquez sur le bouton [+] pour ajouter une ligne à la table. Dans le champ Code, saisissez une variable de contexte afin de sélectionner dynamiquement votre connexion SAP parmi celles prévues dans votre Job.

Lorsqu'un paramètre dynamique est défini, l'option SAPConnection Component List correspondante est désactivée dans la vue Basic settings.

Pour plus d'informations concernant les Dynamic settings et les variables de contexte, consultez le Guide utilisateur Talend Open Studio.

Utilisation

Ce composant est généralement utilisé avec des composants SAP et notamment avec les composants tSAPConnection et tSAPCommit.

Scénario associé

Pour un scénario associé au tSAPRollback, consultez la section intitulée « Scénario : Annuler l’insertion de données dans des tables mère/fille » du composant tMysqlRollback.
