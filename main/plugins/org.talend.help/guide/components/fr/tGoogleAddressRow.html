Avertissement

Ce composant est disponible dans la Palette du studio si vous avez souscrit à l'un des produits Talend Platform.

Propriétés du tGoogleAddressRow

Famille de composant

Data Quality/Address

 

Fonction

Le composant tGoogleAddressRow accède à l'API Google Geocoding via une requête HTTP afin d'obtenir des coordonnées géographiques ainsi que d'autres informations géographiques, selon les informations relatives à l'adresse que vous avez fournies.

Note

Pour plus d'informations concernant l'API Google Geocoding, consultez The Google Geocoding API (en anglais).

Objectif

Le tGoogleAddressRow vous permet de convertir des adresses d'un format lisible par l'homme en coordonnées géographiques et autres informations géographiques.

Basic settings

Pause(in 1/10 seconds)

Temps de pause entre les requêtes HTTP pour les enregistrements relatifs aux informations géographiques. Par défaut, la valeur de ce champ est 1.

 

Schema et Edit Schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

Utilisation

Ce composant est un composant intermédiaire. Il nécessite un composant d'entrée, un composant de sortie et un lien Main Row.

Scénario : Obtention d'informations géographiques détaillées à l'aide d'adresses

Ce scénario décrit un Job a trois composants envoyant une requête HTTP à l'API Google Geocoding et obtenant des informations géographiques détaillées, à partir d'une adresse fournie.

Construire le Job

  1. Déposez les composants suivants de la Palette dans l'espace de modélisation graphique : tFixedFlowInput, tGoogleAddressRow et tLogRow.

  2. Reliez le tFixedFlowInput au tGoogleAddressRow et le tGoogleAddressRow au tLogRow à l'aide de liens Main.

Configurer les composants

  1. Double-cliquez sur le composant tFixedFlowInput afin d'afficher sa vue Basic settings et définir ses propriétés.

  2. Cliquez sur Edit schema pour ouvrir une boîte de dialogue affichant la structure des données d'entrée et définir les données que vous souhaitez passer au composant tGoogleAddressRow. Dans ce scénario, passez deux colonnes, AddressID et Address. Cliquez sur OK pour fermer la boîte de dialogue.

  3. Cliquez sur Use Inline Table dans la zone Mode de la vue Basic settings du tFixedFlowInput puis cliquez sur le bouton [+] pour ajouter une colonne à la table.

  4. Renseignez la colonne AddressID en saisissant 1 et la colonne Address en saisissant "500 Oracle Parkway,Redwood Shores, CA 94065".

  5. Double-cliquez sur le composant tGoogleAddressRow pour afficher sa vue Basic settings et définir ses propriétés.

  6. Sélectionnez Address dans la liste Address Column et laissez les autres paramètres tels qu'ils sont.

  7. Double-cliquez sur le tLogRow pour afficher sa vue Basic settings et définir ses propriétés.

  8. Sélectionnez Vertical (each row is a key/value list) dans la zone Mode et Print unique name dans la zone Title printing mode.

Exécuter le Job

  • Sauvegardez votre Job et appuyez sur F6 pour l'exécuter.

    Note

    Une limite de 2500 demandes de géolocalisation par jour et par adresse IP est appliquée par Google, même si aucune clé pour l'API Google API n'est requise par l'API Geocoding (V3). Pour plus d'informations concernant les limites de l'API Geocoding, consultez la section Usage Limits de la page The Google Geocoding API (en anglais).

Les informations géographiques de l'adresse 500 Oracle Parkway,Redwood Shores, CA 94065 sont obtenues de la base de données Google Map et affichées dans la console.
