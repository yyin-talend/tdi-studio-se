tThresholdViolationAlert

Avertissement

Ce composant est disponible dans la Palette du studio si vous avez souscrit à l'un des produits Talend Platform.

Avertissement

Ce composant est utilisable uniquement avec la base de données de rapports (MySQL) stockant les résultats des analyses exécutées dans le studio. Pour plus d'informations concernant la base de données de rapports, consultez le chapitre relatif aux rapports, dans le Guide utilisateur de Talend Data Quality Studio.

Propriétés du tThresholdViolationAlert

Famille de composant

Data Quality

 

Fonction

Le tThresholdViolationAlert peut alerter en cas de violation de n’importe quel seuil parmi les seuils définis sur les indicateurs, lors d’analyses créées dans la perspective Profiling du studio.

Objectif

Le tThresholdViolationAlert communique avec la base de données de rapports stockant les résultats d'analyse. Il scanne la base de données de rapports afin de trouver les seuils violés dans les analyses de colonnes dans le rapport sélectionné. Vous pouvez l’utiliser avec d’autres composants dans un Job, par exemple le tSendMail, afin d’alerter en cas de violation des seuils définis sur les indicateurs utilisés dans les analyses de colonnes.

Pour ne pas créer un Job à partir de rien, vous pouvez générer un Job prêt à l'emploi utilisant un tThresholdViolationAlert depuis la perspective Profiling de votre studio. Pour plus d’informations, consultez le chapitre des rapports, dans le Guide utilisateur de Talend Data Quality Studio.

Basic settings

Schema et Edit schema

Un schéma est une description de lignes, il définit le nombre de champs qui sont traités et passés au composant suivant. Le schéma est soit local (built-in) soit distant dans le Repository.

 

 

Built-in : Le schéma est créé et conservé ponctuellement pour ce composant seulement. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

 

Repository : Le schéma existe déjà et est stocké dans le Repository. Ainsi, il peut être réutilisable dans divers projets et Job designs. Voir également le Guide utilisateur de Talend Data Integration Studio.

 

Output field

Sélectionnez dans la liste le champ de sortie qui contiendra les données d’entrée.

 

Additional Information

Lorsque le Job est généré, cette zone est pré-renseignée avec des informations détaillées concernant les violations de seuils dans le rapport sélectionné. Le nom du rapport, le nom de l'analyse et la description de l'élément analysé (nom de la colonne, nom de la table et nom du schéma) sont également listés.

Les mêmes informations seront envoyés dans le message (e-mail).

Advanced settings

Advanced separator (for number)

Cochez cette option pour modifier les séparateurs utilisés pour les nombres :

Thousands separator : définissez le séparateur utilisé pour les milliers.

Decimal separator : définissez le séparateur utilisé pour les décimaux.

 

Separator between Suffix and Field Value

Saisissez le caractère qui séparera le nom du champ de sa valeur.

 

Field Separator

Caractère, chaîne ou expression régulière séparant les champs.

 

Row Separator

Chaîne (ex : “\n” sous Unix) séparant les lignes.

 

tStatCatcher Statistics

Cochez cette case pour collecter les métadonnées de process du Job, aussi bien au niveau du Job qu’au niveau de chaque composant.

Utilisation

Ce composant est un composant intermédiaire. Il requiert des composants d’entrée et de sortie. Le tThresholdViolationAlert ne fonctionne qu’avec une base de données MySQL.

Scénario associé

Aucun scénario n’est disponible pour ce composant.
