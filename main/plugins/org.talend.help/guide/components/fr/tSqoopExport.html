tSqoopExport

Avertissement

Ce composant est disponible dans la Palette du studio uniquement si vous avez souscrit à l’édition correspondante de Talend Big Data Studio.

Propriétés du tSqoopExport

Famille de composant

Big Data/Sqoop

 

Fonction

Le tSqoopExport appelle Sqoop pour transférer des données du système de fichiers distribué Hadoop (Hadoop Distributed File System, HDFS) vers un système de gestion de base de données relationnelle (SGBDR).

Note

Assurez-vous d'avoir installé Sqoop sur l'hôte utilisé pour exécuter ce composant. Assurez-vous également d'ajouter l'invite de commande Sqoop à la variable PATH de cet hôte.

Objectif

Le tSqoopExport est utilisé afin de définir les arguments requis par Sqoop pour transférer des données dans un SGBDR.

Basic settings

Connection

Saisissez l’URL JDBC utilisée pour vous connecter à la base de données cible.

 

Table Name

Saisissez le nom de la table cible dans laquelle les données sont transférées depuis HDFS. Cette table doit déjà exister dans la base de données cible. Les fichiers d’entrée sont lus et analysés dans un ensemble d’enregistrements selon les délimiteurs personnalisés.

 

Export Dir

Saisissez le chemin d’accès ou parcourez votre répertoire jusqu’aux données sources devant être transférées dans HDFS.

 

User name et Password

Saisissez vos informations d’authentification à la base de données cible.

 

Specify Number of Mappers

Cochez cette case afin de préciser le nombre de tâches de “map” (processus parallèles) utilisé pour effectuer le transfert des données.

Si ne vous souhaitez pas que Sqoop travaille en parallèle, saisissez 1 dans le champ affiché.

 

Print Log

Cochez cette case pour activer la case Verbose.

 

Verbose

Cochez cette case pour imprimer plus d’informations pendant que vous travaillez, par exemple des informations de débogage.

Advanced settings

Direct

Cochez cette case pour utiliser le chemin d’export rapide.

 

Use MySQL default delimiters

Cochez cette case pour utiliser les délimiteurs MySQL par défaut.

 

Additional arguments

Complétez ce tableau pour utiliser des arguments supplémentaires, si nécessaire.

En ajoutant des arguments supplémentaires, vous pouvez effectuer de multiples opérations en une seule transaction. Par exemple, vous pouvez utiliser --hive-import et --hive-table pour créer une table Hive et écrire des données dans cette table lors de l'exécution de la transaction écrivant des données dans HDFS. Pour plus d'informations concernant les arguments Sqoop disponibles, consultez le manuel de Sqoop.

 

tStatCatcher Statistics

Cochez cette case pour collecter les données de log au niveau des composants.

Utilisation

Ce composant est utilisé en standalone. Il respecte les prérequis Sqoop. Vous devez connaître Sqoop pour l'utiliser.

Il est recommandé d'utiliser une version 1.4+ de Sqoop afin de tirer entièrement parti des fonctionnalités des composants.

Pour plus d'informations concernant Sqoop, consultez (en anglais) le manuel de Sqoop sur le site : http://sqoop.apache.org/docs/1.4.0-incubating/SqoopUserGuide.html

Limitation

Vous devez vous assurer que l'hôte, Sqoop et le Job sont sur la même machine, pour exécuter le Job utilisant ce composant.

Si le Studio n'est pas déployé dans l'hôte, vous pouvez installer le serveur de Job fourni par Talend sur l'hôte et utiliser le serveur pour exécuter le Job.

Connections

 

Liens de sortie (d’un composant à un autre) :

Trigger : Run if, On Subjob Ok, On Subjob Error.

Liens d’entrée (d’un composant à un autre) :

Pour plus d’informations concernant les connexions, consultez le Guide utilisateur de Talend Data Integration Studio.
